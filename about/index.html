About Beaker

An experimental browser

Beaker is an experimental browser built to demonstrate an alternative vison for the Web.

Host peer-to-peer websites
Fork and edit applications
Share files privately

Why

The Web is losing its spirit of freedom and innovation by depending on a small set of centrally-operated services. We believe this can be solved with new protocols and application models.

Service-based applications are closed-source and privately owned. By inverting the model to run “thick” clientside applications and depending only on peer-to-peer protocols and fungible thin services, we can empower users to communally modify source code, control their identities, and secure their data.

We envision a future where communities can build applications in response to their own needs. New forks of an application could go viral weekly, and the community would be actively engaged in the forks they prefer.

How it works

Beaker uses Dat, a peer-to-peer protocol to manage and sync data stores. Sites published with Beaker are:

  • Secure - Changes are written to an append-only log and content is verified upon arrival, creating a version history that improves auditability.
  • Distributed - Users connect directly to other users to retrieve content. Anyone can host files without depending on a centralized service.
  • User-owned - Data and source code live on the user’s device and run in a secure, configurable sandbox. Services are optional.
